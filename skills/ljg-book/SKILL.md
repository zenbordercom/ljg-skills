# ljg-book: 拆书

输入一本书，输出它的骨架。

## 拆书是什么

不是读后感，不是章节摘要，不是夸——是把作者的骨架抽出来摆桌上看。

读一本书容易迷在作者的笔触里。拆一本书是从笔触里跳出来，看作者的取景框。取景框看清，整本书才看清。

## 五件事

每一件单独问一遍。问完串起来。

### 1. 核心问题 + 挠痒处

作者真正在答什么？而且——他为什么不写不行？

题目是 topic。问题是 question。挠痒处是 motive——三层都要，缺了挠痒处就是空。

挠痒处具体到什么程度：

- 作者看到什么具体的痛/愤怒/焦虑/兴奋？
- 他对读者目前的态度有什么不满？
- 他在跟谁说话？（哪类读者、哪种立场）
- 不写会怎样？

例子：拆 KK《必然》不能停在「他想找科技演化的规律」——要写到「他看到一群人在恐惧 AI、抵抗新科技、想冻住工业时代，他要告诉这些人：这些变化是 inevitable 的，焦虑没用」。书名往往就是答案的种子——「必然」两个字是在跟恐惧的人说话。

怎么找：读作者的导言/前言，找"我写这本书是为了……"那一句。读不到，看作者反复回到的那一处——他绕了几百页都甩不掉的东西，就是挠痒处本身。

写完测试：这段拿去描述同领域另一本书，还成立吗？成立 → 太抽象，重写到具体。

### 2. 基础假设

作者立在什么不证之物上？

每本书都有几条天花板——作者不去证它们，他靠它们站着。动摇任何一条，整本书就塌。

操作问句两条，对着书问：
- 这书里什么是不许问的？（一问就要被作者归为外行）
- 什么是作者要相信才能往下写的？（少了它，后面所有论证失支点）

答出来的几条，就是这本书的基础假设。

亮出来不展开。这几条本身不证——摆桌上就行，证不证不在拆书人手里。

### 3. 分析框架

作者用什么看世界？

每个作者都有一台取景框。Darwin 的取景框是"变异 + 选择 + 时间"——他看蝴蝶看人看部落都是这台框。Marx 的取景框是"生产关系决定上层建筑"——他看历史看社会看艺术都是这台框。Freud 的取景框是"意识冰山一角，水下才是真主角"——他看梦看口误看政治都是这台框。

找作者的框：他反复用什么工具？他从什么角度切入？他看到 X 第一反应想到的是 Y——这个 Y 就是他的框。

框不一定是名词。可以是一个动作（"还原到 root cause"）、一个区分（"自然态 vs 文化态"）、一个对照（"显性逻辑 vs 隐性结构"）。

找框要够狠——一句话讲清楚，不是一个章节。

**强制：必须列出作者独占的术语/列表/区分。**

每本好书都有作者独占的「标签」——别人没有、他独有、能 quote。这是拆书的"指纹"。少了它，整段拆书可以套到任何同领域的书上。

- KK《必然》必须有：12 个动词列表（Becoming / Cognifying / Flowing / Screening / Accessing / Sharing / Filtering / Remixing / Interacting / Tracking / Questioning / Beginning）+ technium + protopia + becoming vs 完成
- Marx《资本论》必须有：生产关系 / 上层建筑 / 剩余价值 / 阶级
- Kahneman《思考快与慢》必须有：System 1 / System 2 / 锚定 / 损失厌恶
- 海德格尔《存在与时间》必须有：Dasein / 上手性 / 此在的时间性

拆完看一眼：作者独占的标签出现了几个？0 个 → 你拆的不是这本书，是这本书所属的领域。

### 4. 核心观点 / 结论

作者最后得出了什么？

不是"他举了多少例子"，不是"他论证了多少环节"。是：合上书，作者要你带走的那个东西。

往往就一两句话。

检验：把这句话单独抽出来，脱离作者上下文，它还有没有力量？如果只是复述书名，那不是核心观点——是 label。如果只是套话（"事物都是辩证的""万物相联"），那是空话。

核心观点要锋利——能立得住，能反驳，能被引用。

不能软。"作者认为人性是复杂的"是软话——什么不是复杂的？

另问一遍：**作者最想让读者带走的那一句话是什么？**

能直接 quote 最好，quote 不到用最贴近的复述。这一句要锋利到——读者合上书 10 年后还能想起来。

### 5. 精神内核 — 带走一件

合上书 10 年后只能记一件事——是哪一件？

不是把前 4 段缩短。那是总结，是重复。这一件是从整本书萃出的内核：单独立住、可携带、能让人想回去翻原书。

形态五选一。挑作者最依赖的那一种，不堆，不并列：

- **一个取景框**——一种看世界的方式
  - Darwin：变异 + 选择 + 时间
  - Freud：意识冰山，水下才是主角
  - KK《必然》：科技演化的 inevitability + 12 动词
- **一个模型**——一个结构性认知工具
  - Kahneman：System 1 / System 2
  - Marx：经济基础 / 上层建筑
  - Conant-Ashby：调节器必含被调节系统的内模型
- **一个洞见**——一句被作者撬开的真相
  - "组织的命运不是被战略决定，是被结构决定。"
  - "我们以为是选择，其实是默认。"
- **一个概念**——作者铸的新词
  - Taleb：反脆弱 / 黑天鹅
  - 海德格尔：Dasein
  - KK：technium
- **一个金句**——作者亲笔写出、可直接 quote
  - "All happy families are alike; each unhappy family is unhappy in its own way."
  - "The medium is the message."

怎么判作者最依赖哪一种？看他书里反复回到的是什么——一台看世界的眼镜（取景框）、一对概念区分（模型）、一个被点破的事实（洞见）、一个他造的词（概念），还是一句他亲笔写下的话（金句）。

三行内交付。形态标签 + 内容本体 + 一句话点为什么是这一件。

挑完三检：
- 合上书 10 年只能记一件事，就是这件吗？是 → 过。不是 → 没挑到内核，重挑。
- 这件事跟前 4 段重了吗？重了 → 这是总结不是内核，重挑。
- 一个没读过的人，听了这一件，会不会想去翻原书？会 → 抓到。不会 → 太软或太抽象。

收尾一句：带走了这一件，这本书的精神内核就在你手里。

## 灵魂三问 + 指纹测试

写完整篇拆书，默问自己三个问题。任何一个答不出 → 没拆到位，回头补：

1. **作者「不写不行」的那股劲儿是什么？** 不是 topic，是他看见什么具体的痛/机会/愤怒/焦虑——能 quote 出他的动机。
2. **作者最独占的那个 move 是什么？** 能 quote 的具体动作 / 概念 / 列表 / 区分。别人没有、他独有。
3. **作者想让读者带走的那一句话是什么？** 锋利到 10 年后还记得。

**灵魂 = 第 1 问。精髓 = 第 2 问。压缩 = 第 3 问。三个都答得出 → 灵魂和精髓抓住了。第 5 模块「精神内核」是把这些聚成单一形态——只挑一件带走，作为整本书的指纹。**

每段写完做两道关卡：

- **替换测试**：把这段拿去描述同领域另一本书，还成立吗？KK《必然》的拆能套到 Harari《未来简史》上吗？能套 → 太抽象，杀掉重写。不能套 → 抓到了这本书的指纹。
- **指纹测试**：这段里有几个只有这本书才有的术语/概念/列表/区分？0 个 → 你拆的是"领域"不是"这本书"。1-3 个 → 及格。3+ 个 → 好。

## 怎么写

冷静、简洁、犀利、直白。

不夸作者，不贬作者，不替作者辩护，不替作者道歉。

五段，标号 1-5。段内一气呵成，不绕弯。

每段开头一句立住——这一段要 say 的核心是什么。然后展开两三句把它讲透。不要列点列到天荒地老——一段散文胜过十个 bullet。

短句多。介词砍。"通过 / 借 / 凭 / 在 X 上"——能不用就不用。

中文白话——汪曾祺、王小波、阿城那一路。每写完一段默念出声，卡的地方整句重写（不是抠字）。

不要学术腔。"作者论证了""作者认为"少用——直接说作者干了什么。"X 在第 N 章指出……" 全部杀掉。

## 反翻译腔自查（每段写完扫一遍）

英文母语模型用中文最容易犯的毛病。信号词出现 → 整句重写。

**词汇直译信号**：

| 翻译腔 | 真实中文 |
|--------|---------|
| 我接住你 | 我在 |
| 我不装 | 直说；不绕弯 |
| 给你拥抱（文字上的） | 抱抱；过来 |
| 你值得更好的 | 你不该这样 |
| 让我们一起... | 我们...（省「让」） |
| 提供支持 | 撑你 |
| 在情感上 X | 情感上 X（省「在」） |
| 这是必要的 | 必须这样；非这样不可 |
| 做出选择 / 做出决定 | 选；决定 |
| 在这个意义上 | 这么说 |
| 拥有...的能力 | 能... |
| 进行（研究/讨论/对话） | 直接用具体动词 |
| 这片领域 | 这门学问；这套东西 |
| 通过 X 抓到 Y | X 动词直接带宾 |
| 一是...二是...三是... | 散文式排开，不编号 |

**句式英文化信号**：
- 被动句滥用：「被这事困扰」→「这事缠着」
- "是...的"句：「这是重要的」→「这事要紧」
- 长定语：「那个由 X 引起的 Y」→ 拆两句
- 括号补充：「拥抱（文字上的）」整段删，换具体描述
- "在 X 上"滥用：「在结构上」→「结构上」或改写
- 三层定语：「X 的 Y 的 Z」→ 拆两句（一个「的」做定语没事，两个起停下来）

**词性英文化信号**：
- 动名词化：「做出决定」→「决定」；「展开对话」→「聊起来」
- 抽象集合名词偏好：「领域 / 阶段 / 过程 / 动作 / 方式 / 模式 / 现象」→ 换具体（「这门 / 这段 / 这趟 / 这件事 / 怎么说 / 这种样子 / 这事」）
- 形容词副词堆砌：「深刻的、有意义的、重要的」→ 选一个
- 介词链滥用：「通过 / 借 / 在 X 上 / X 之间 / X 之中」→ 动词直接带宾

**中文意合三省**：
- 连词省（「因为...所以」→ 直接说后半句）
- 主语省（上下文能补的别重复）
- 「的」省（「温柔的支持」→「温柔地撑」）

**朗读关**：信号词 grep 抓不到的 frame 级翻译腔，靠默念出声捕。每段默念一遍——卡的地方整句重写，不是抠字。问自己一句：「汪曾祺会这么写吗？」

## 怎么找材料

如果用户只给书名——
- 先 WebSearch / WebFetch 找作者、导言、目录、核心论点的二手解读
- 拿到核心 argument 后再答
- 不要凭印象瞎拆——印象拆出来是 wikipedia 风的空话

如果用户给了 PDF / arxiv / 内容——直接读，不用查。

如果是经典书——可以靠 LLM 内部知识，但仍要在心里默问：我说的这条是书里实际有的，还是我编的？编的就删。

## 输出

1. 获取时间戳：`date +%Y%m%dT%H%M%S` 和 `date "+%Y-%m-%d %a %H:%M"`
2. 写入 `~/Documents/notes/{时间戳}--拆书-{书名}__book.org`
3. org-mode 格式（标题用 `*`，加粗用 `*bold*` 单星号，禁止 markdown 双星号）
4. 文件头：

```
#+TITLE: 拆书：《{书名}》
#+SUBTITLE: {作者} | {一句话核心观点}
#+DATE: [{YYYY-MM-DD Day HH:MM}]
#+FILETAGS: :book:{书的领域 e.g. philosophy/biology/economics}:
#+IDENTIFIER: {YYYYMMDDTHHMMSS}
```

5. 正文五段，标 1-5，每段散文。
6. 报告文件路径给用户。

## 红线（每条必过）

1. **不夸不贬不替辩护** — 拆书人站作者头上看，不是粉丝也不是反对者
2. **冷静、简洁、犀利、直白** — 每段读出声不卡
3. **核心观点不能软** — 软话（"X 是复杂的"）杀掉
4. **基础假设不展开论证** — 亮出来就行
5. **精神内核挑一种形态** — 取景框/模型/洞见/概念/金句五选一，三行内交付，不重前 4 段
6. **零学术腔** — "作者论证了 / 认为 / 指出"少用
7. **反翻译腔** — 内嵌「反翻译腔自查」清单全过；信号词清空；写完默念汪曾祺会这么写吗
8. **必须含作者独占术语/列表/区分** — KK 拆书少了 12 动词、Marx 拆书少了生产关系，就是失职
9. **灵魂三问必答** — 「不写不行的劲儿」/「最独占的 move」/「那一句」三问任何一个答不出，回头补
10. **替换测试 + 指纹测试** — 每段写完都过这两道关，套得到别的书 = 杀掉重写
